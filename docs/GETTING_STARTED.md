# 🚀 Быстрый старт

## Требования

- `Python 3.11+`
- `PostgreSQL 15+`
- `Redis 7+`
- `Telegram Bot Token`

## Установка

```bash
git clone <repository-url>
cd telegram-shift-bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## `.env`

```env
BOT_TOKEN=your_telegram_bot_token_here
ADMIN_IDS=123456789,987654321

# PostgreSQL
DB_NAME=shift_bot
DB_USER=bot_user
DB_HOST=postgres
DB_PORT=5432
DB_PASSWORD=change_me

# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=change_me
REDIS_DB=0

# Security
ENCRYPTION_KEY=replace_with_your_key

TZ=Europe/Moscow
LOG_LEVEL=INFO

ENABLE_VERIFICATION=False
ENABLE_GROUP_REMINDERS=True

POLL_CREATION_HOUR=9
POLL_CREATION_MINUTE=0
POLL_CLOSING_HOUR=19
POLL_CLOSING_MINUTE=0
REMINDER_HOURS=[17]
```

## Запуск инфраструктуры

```bash
cp .env.example .env
docker compose up -d postgres redis
python3 scripts/init_runtime_database.py
python3 src/main.py
```

Если PostgreSQL и Redis запущены через `docker compose`, бот при запуске с хоста сам пробует подключение как к Docker-именам (`postgres`, `redis`), так и к `localhost`.

Если хотите запускать весь проект через Docker:

```bash
docker compose up -d
```

## Базовый сценарий настройки

1. Добавьте бота в группу ЗИЗ.
2. Откройте админку через кнопку `Админ-панель` снизу или через `/admin`.
3. Настройте слоты для дневных групп. Для ночной группы отдельные временные слоты не нужны.
4. Заведите сотрудников через раздел `Сотрудники`.
5. При необходимости откройте `Сотрудники` → `Статус привязки Telegram`, чтобы проверить, кто уже связался с Telegram ID после голосования.
6. Если Telegram подтянул некорректное имя, используйте `Сотрудники` → `Переименовать сотрудника` и сохраните правильные `Имя Фамилия`.
7. Если сотрудник перешел на другой ЗИЗ, используйте `Сотрудники` → `Перенести в другую группу`.
8. Если сотрудник уже привязан к Telegram и впервые проголосует в новой группе, бот перенесет его автоматически.
9. Дождитесь автоматического цикла:
- `09:00` — создание опросов
- `17:00` — напоминание неотметившимся
- `19:00` — итог и список тех, кто не отметился для дневных групп
- `17:00` — итог для ночных групп

## Быстрый тест вручную

1. Откройте `/admin` → `Опросы` → `Отправить тестовый опрос`.
2. Выберите нужную тестовую группу.
3. Проголосуйте в опросе хотя бы с одного аккаунта.
4. Откройте `/admin` → `Опросы` → `Результаты опросов`.
5. Проверьте, что бот показывает:
- проголосовавших по выходам;
- `Куратор`, если такой вариант выбран;
- `Выходной`, если такой вариант выбран;
- список сотрудников, которые не отметились.
6. Откройте `/admin` → `Опросы` → `Закрыть тестовый опрос`, чтобы завершить тест вручную.
7. При ручном закрытии бот сразу отправляет в чат итог и список тех, кто не отметился.
8. Для проверки напоминания откройте `/admin` → `Опросы` → `Тест напоминания 17:00`.
9. Если нужно полностью очистить тесты группы, откройте `/admin` → `Опросы` → `Удалить все тестовые опросы` и подтвердите действие.
10. Если активных тестовых опросов несколько, при закрытии бот покажет список и даст выбрать нужный.

## Что важно помнить

- Внутренних тем в группах нет: опросы, рассылки, напоминания и итоги идут прямо в общий чат группы.
- Дневной опрос называется `Выходы на завтра`.
- Ночной опрос называется `Выходы на сегодня`.
- При создании опрос закрепляется в группе с уведомлением участников.
- Имена в итогах берутся из реестра сотрудников, если администратор заранее их поправил.
- Для новой дневной группы стандартные слоты подставляются автоматически: `07:00-19:00`, `08:00-20:00`, `09:00-21:00`, `10:00-22:00`, `12:00-00:00`.
- Для новой ночной группы бот автоматически ставит ночной сценарий без дневных слотов и с закрытием в `17:00`.

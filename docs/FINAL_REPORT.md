   ├─ Forbidden content
   └─ Matched source

   └─ Логирование
   ↓

## 📊 УЛУЧШЕНИЯ

### Must-Match проверка

**Было:**
- Требует ВСЕ триггеры одновременно
- Пропускает валидные кейсы

**Стало:**
- Считает совпадения триггеров (score)
- Приоритеты (P0 > P1 > P2)
- Threshold 0.8
- Находит больше валидных кейсов

### Confidence Routing

**Было:**
- Простая проверка наличия FAQ
- Нет реального расчета score

**Стало:**
- Реальный расчет confidence score по формуле:
  ```
  score = must_match_found * 1.0 + 
          faq_relevance * 0.7 + 
          tag_confidence * 0.6 + 
          rule_clarity * 0.5
  ```
- Пороги: 0.8 (auto_answer), 0.6 (clarification), 0.0 (route_to_curator)
- Проверка обязательной эскалации

### Валидация ответа

**Было:**
- Нет валидации перед отправкой

**Стало:**
- Проверка обязательных блоков
- Проверка тега
- Проверка forbidden content
- Проверка matched_source

### Explainability Logs

**Было:**
- Нет структурированного логирования

**Стало:**
- Все решения логируются в структурированном виде
- Сохранение в JSONL файлы (по дням)
- Все поля из explainability_log_schema

### Автотесты

**Было:**
- Нет автотестов

**Стало:**
- Автотесты по Golden Set
- Проверка decision_route, тега, confidence
- Параметризованные тесты

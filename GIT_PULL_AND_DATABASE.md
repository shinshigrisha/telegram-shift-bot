# ⚠️ GIT PULL И БАЗА ДАННЫХ

## ❌ КОРОТКИЙ ОТВЕТ

**НЕТ, база данных НЕ переносится через `git pull`.**

`git pull` переносит только:
- ✅ Код приложения
- ✅ Миграции SQL (структуру БД)
- ✅ Конфигурационные файлы

**НЕ переносится:**
- ❌ Данные таблиц (FAQ, пользователи, опросы)
- ❌ Docker volumes (где хранятся данные БД)

---

## ✅ ЧТО ДЕЛАТЬ

### Вариант 1: Обновление кода (данные уже есть на сервере)

Если на сервере уже есть БД с данными:

```bash
# На сервере
cd /opt/telegram-shift-bot
git pull
docker compose restart bot
```

**Данные сохранятся**, обновится только код.

---

### Вариант 2: Перенос данных с локального компьютера

Если нужно перенести данные из локальной БД на сервер:

#### 1. На локальном компьютере: создать бэкап

```bash
# Бэкап PostgreSQL
docker compose exec postgres pg_dump -U bot_user shift_bot | gzip > backup.sql.gz
```

#### 2. Скопировать бэкап на сервер

```bash
scp backup.sql.gz root@194.87.250.251:/opt/telegram-shift-bot/backups/
```

#### 3. На сервере: восстановить данные

```bash
cd /opt/telegram-shift-bot
git pull  # Обновить код

# Восстановить данные
gunzip -c backups/backup.sql.gz | docker exec -i telegram-shift-postgres psql -U bot_user -d shift_bot
```

---

## 📋 БЫСТРАЯ ИНСТРУКЦИЯ

### Если на сервере уже есть БД:
```bash
git pull && docker compose restart bot
```

### Если нужно перенести данные:
1. Создать бэкап на локальном компьютере
2. Скопировать на сервер
3. Восстановить из бэкапа

---
